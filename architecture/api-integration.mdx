All network requests are centralised in a single module: `src/api/transactions.ts`. No component or context calls `fetch` directly — they import named functions from this file.

## Base URL

The backend origin is read from an environment variable at build time:

```ts
const API_URL = import.meta.env.VITE_API_URL;
```

Set this in a `.env` file at the project root:

```bash
VITE_API_URL=http://localhost:3000
```

<Note>
  Vite only exposes variables prefixed with `VITE_` to client-side code. Any variable without this prefix will be `undefined` at runtime.
</Note>

## Passing filters

Functions that accept optional filters convert the plain object to a `URLSearchParams` instance before appending it to the URL:

```ts
export async function getTransactions(filters = {}): Promise<Transaction[]> {
  const params = new URLSearchParams(filters);
  const res = await fetch(`${API_URL}/api/transactions?${params}`);
  if (!res.ok) throw new Error(`Error loading transactions: ${res.status}`);
  return res.json();
}
```

The `Filters` type used across the app is:

```ts
export type Filters = {
  type?:     string;
  category?: string;
  year?:     string;
  month?:    string;
}
```

## Error handling

Every function checks `res.ok` (HTTP status 200–299). On failure it throws an `Error` containing the HTTP status code. For mutation endpoints (POST/PATCH/PUT/DELETE) the response body is also parsed for a server-supplied `message`:

```ts
if (!response.ok) {
  const data = await response.json().catch(() => ({}));
  throw new Error(
    data.message ?? `Error adding transaction (${response.status})`
  );
}
```

Callers (contexts and hooks) catch these errors and store the message in their `error: string | null` state, which components can then display.

```ts createTransaction
export async function createTransaction(
  newItem: Transaction,
): Promise<Transaction> {
  const response = await fetch(`${API_URL}/api/transactions`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(newItem),
  });
  if (!response.ok) {
    const data = await response.json().catch(() => ({}));
    throw new Error(data.message ?? `Error adding transaction (${response.status})`);
  }
  return response.json();
}
```

```ts updateTransaction
export async function updateTransaction(
  item: Transaction,
): Promise<Transaction> {
  const response = await fetch(`${API_URL}/api/transactions/${item.id}`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(item),
  });
  if (!response.ok) {
    const data = await response.json().catch(() => ({}));
    throw new Error(data.message ?? `Error editing transaction (${response.status})`);
  }
  return response.json();
}
```
