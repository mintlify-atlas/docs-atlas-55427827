The Personal Finance frontend is a React single-page application built with TypeScript and bundled by Vite. It communicates with a REST API backend over HTTP and runs entirely in the browser.

## Request flow

| Layer | Description |
|---|---|
| **Client browser** | Renders the React SPA, manages routing client-side |
| **React SPA** | Handles UI, state, and all user interaction |
| **REST API backend** | Provides data via JSON endpoints; base URL set in `VITE_API_URL` |

The SPA never serves its own data — every read and write goes through the API.

## Folder structure

| Directory | Purpose |
|---|---|
| `src/api/` | All fetch calls to the backend REST API |
| `src/components/` | Shared UI components (Aside, Footer, ProtectedRoute, etc.) |
| `src/context/` | React Context providers for global state |
| `src/hooks/` | Custom hooks for data fetching, pagination, and form state |
| `src/pages/` | Top-level page components rendered by the router |
| `src/types/` | Shared TypeScript type and interface definitions |
| `src/utils/` | Utility/helper functions |

## Page routing

Routing is handled by **React Router DOM v7**. Routes are defined in `src/App.tsx`.

| Path | Component | Protected |
|---|---|---|
| `/login` | `Login` | No |
| `/` | `Dashboard` | Yes |
| `/categories` | `Categories` | Yes |
| `/transactions` | `Transactions` | Yes |
| `/settings` | `Settings` | Yes |
| `*` | `NotFound` | No |

All authenticated routes are wrapped inside a single `<ProtectedRoute />` element:

```tsx
<Route element={<ProtectedRoute />}>
  <Route path="/" element={
    <DashboardContextProvider>
      <Dashboard />
    </DashboardContextProvider>
  } />
  <Route path="/categories" element={<Categories />} />
  <Route path="/transactions" element={
    <TransactionsContextProvider>
      <Transactions />
    </TransactionsContextProvider>
  } />
  <Route path="/settings" element={<Settings />} />
</Route>
```

## Authentication

`ProtectedRoute` (`src/components/ProtectedRoute.tsx`) reads the `user` value from `SettingsContext`. If `user` is `null`, it redirects to `/login`. Otherwise it renders the child routes via `<Outlet />`.

```tsx
export default function ProtectedRoute() {
  const { user } = useSettings()

  if (!user) {
    return <Navigate to={"/login"} replace />
  }

  return <Outlet />;
}
```

User session data is persisted to `localStorage` under the key `user-login-finances` (see [State management](/architecture/state-management)).

## Lazy loading

Every page component is code-split using `React.lazy` and wrapped in a single top-level `<Suspense>` boundary with a `LoadingFallback`.

```tsx
const Dashboard     = lazy(() => import("./pages/Dashboard"))
const Transactions  = lazy(() => import("./pages/Transactions"))
const Categories    = lazy(() => import("./pages/Categories"))
const Settings      = lazy(() => import("./pages/Settings"))
const Login         = lazy(() => import("./pages/Login"))
const NotFound      = lazy(() => import("./pages/NotFound"))
```

This ensures each page's JavaScript is only downloaded when the user first navigates to it.
