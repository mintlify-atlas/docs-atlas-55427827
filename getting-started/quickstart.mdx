## Prerequisites

Before you begin, make sure you have the following:

- **Node.js** v18 or later
- **npm** v9 or later (bundled with Node.js)
- A running **backend API** that the frontend can connect to

<Note>
Personal Finance Web is a frontend-only repository. It requires a compatible REST API backend to fetch and persist financial data. Have your backend URL ready before starting.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine and navigate into the directory.

    ```bash
    git clone https://github.com/your-org/finance-api-frontend.git
    cd finance-api-frontend
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager.

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash yarn
    yarn install
    ```

    ```bash pnpm
    pnpm install
    ```

    </CodeGroup>

    This installs React 19, Vite 7, Tailwind CSS 4, React Router DOM 7, ECharts, and all dev dependencies listed in `package.json`.
  </Step>

  <Step title="Set up environment variables">
    Create a `.env` file in the project root and set the URL of your backend API.

    ```bash .env
    VITE_API_URL=http://localhost:3000
    ```

    The app reads this value at build time and runtime via `import.meta.env.VITE_API_URL`. All API requests in `src/api/transactions.ts` are prefixed with this base URL.

    <Warning>
    All Vite environment variables exposed to the browser must be prefixed with `VITE_`. Variables without this prefix are not available in the frontend bundle.
    </Warning>
  </Step>

  <Step title="Start the development server">
    Run the dev server. The `--host` flag (included in the `dev` script) makes the app accessible on your local network.

    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```

    </CodeGroup>

    Vite will print the local URL, typically `http://localhost:5173`. Open it in your browser.
  </Step>

  <Step title="Create your account and log in">
    The app opens at `/login`. No external authentication service is required — the login flow is entirely local:

    1. Click the avatar to open the avatar picker and choose one of the ten available DiceBear avatars.
    2. Enter a username in the **Username** field.
    3. Click **Create Account**.

    Your session is saved to `localStorage` under the key `user-login-finances` as a JSON object:

    ```json
    { "userName": "Alice", "userAvatar": "https://api.dicebear.com/..." }
    ```

    The app reads this value on startup. If the key exists, you are considered logged in and redirected to the Dashboard. If it is absent, you are redirected to `/login`.

    <Tip>
    You can update your display name and avatar at any time from the **Settings** page without losing your session.
    </Tip>
  </Step>

  <Step title="Explore the Dashboard">
    After logging in you land on the Dashboard at `/`. From here you can:

    - Review your current **income**, **expenses**, and **net balance** in the summary cards.
    - View spending breakdowns in the interactive **ECharts** visualizations.
    - Navigate to **Transactions** (`/transactions`) to add, edit, or delete records.
    - Navigate to **Categories** (`/categories`) to manage your spending categories.
    - Open **Settings** (`/settings`) to update your profile or toggle dark mode.
  </Step>
</Steps>
