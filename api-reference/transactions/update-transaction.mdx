---
title: "Update transaction"
description: "Update an existing transaction by ID."
---

## Endpoint

```
PATCH /api/transactions/:id
```

Partially or fully updates an existing transaction. The transaction `id` is passed as a path parameter.

In the frontend, this is called via `updateTransaction(item)` in `src/api/transactions.ts`:

```ts
export async function updateTransaction(
  item: Transaction,
): Promise<Transaction> {
  const response = await fetch(`${API_URL}/api/transactions/${item.id}`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(item),
  });

  if (!response.ok) {
    const data = await response.json().catch(() => ({}));
    throw new Error(
      data.message ?? `Error editing transaction (${response.status})`,
    );
  }

  return response.json();
}
```

## Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the transaction to update.
</ParamField>

## Request body

Send a JSON object containing the fields to update. All `Transaction` fields are accepted; only provided fields need to be included.

<ParamField body="type" type="string">
  Transaction type. One of `income` or `expense`.
</ParamField>

<ParamField body="category" type="string">
  Category name for the transaction.
</ParamField>

<ParamField body="description" type="string">
  Description of the transaction.
</ParamField>

<ParamField body="date" type="string">
  Date of the transaction in ISO 8601 format (e.g. `2025-03-15`).
</ParamField>

<ParamField body="amount" type="number">
  Monetary value of the transaction.
</ParamField>

## Response

Returns the full updated `Transaction` object.

<ResponseField name="id" type="string">
  Unique identifier of the transaction.
</ResponseField>

<ResponseField name="type" type="string">
  Transaction type: `income` or `expense`.
</ResponseField>

<ResponseField name="category" type="string">
  Category of the transaction.
</ResponseField>

<ResponseField name="description" type="string">
  Description of the transaction.
</ResponseField>

<ResponseField name="date" type="string">
  Date of the transaction.
</ResponseField>

<ResponseField name="amount" type="number">
  Monetary value of the transaction.
</ResponseField>

## Example

<CodeGroup>

```bash curl
curl -X PATCH https://your-api.example.com/api/transactions/txn_abc123 \
  -H "Content-Type: application/json" \
  -d '{
    "description": "Updated grocery description",
    "amount": 110.00
  }'
```

</CodeGroup>

### Example response

```json
{
  "id": "txn_abc123",
  "type": "expense",
  "category": "Groceries",
  "description": "Updated grocery description",
  "date": "2025-03-15",
  "amount": 110.00
}
```

<Note>
  The frontend always sends the full `Transaction` object (including `id`) in the request body, even though `id` is a path parameter. The API uses the path parameter as the authoritative identifier.
</Note>
