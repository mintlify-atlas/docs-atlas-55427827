---
title: 'Update category'
description: 'Replace an existing category with new data.'
api: 'PUT /api/categories/{id}'
---

Fully replaces the category identified by `id`. The request body must include both `id` and `name`.

<Warning>
  This endpoint uses **PUT** (full replace), not PATCH. You must supply all fields — omitting `name` may result in an error or empty value.
</Warning>

## Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the category to update.
</ParamField>

## Body

<ParamField body="id" type="string" required>
  The unique identifier of the category. Must match the path parameter.
</ParamField>

<ParamField body="name" type="string" required>
  The new display name for the category.
</ParamField>

## Response

<ResponseField name="id" type="string" required>
  The identifier of the updated category.
</ResponseField>

<ResponseField name="name" type="string" required>
  The updated name of the category.
</ResponseField>

<RequestExample>

```bash cURL
curl --request PUT \
  --url "${VITE_API_URL}/api/categories/4" \
  --header 'Content-Type: application/json' \
  --data '{
    "id": "4",
    "name": "Bills & Utilities"
  }'
```

</RequestExample>

<ResponseExample>

```json 200
{
  "id": "4",
  "name": "Bills & Utilities"
}
```

```json 404
{
  "message": "Category not found"
}
```

</ResponseExample>
